Install

NPM

Install from npm: npm install normform --save

NormForm will be available to import into your project using ES2015 module syntax:

import normform from 'normform';

Download

Grab a distribution copy of normform.min.js. Reference it before the closing <body> tag at the end of your document, or ensure it has an async attribute:

<script src="/path/to/normform.min.js" async><script>

Usage

NormForm is self-initialising, so no need to do any scripting tomfoolery beyond the above.

Get started

First, author a world-class HTML form with the appropriate elements and attributes. Add the js-normform class to the form tag to get things going:

<form class="js-normform">
	...
</form>

Validating inputs

Inputs that need validation are automatically handled by NormForm.

Include an empty <span> element alongside each input. This will become the container to hold the validation feedback. This is bound to the relevant input by including the data-normform-feedback-for attribute. Its value needs to match the input id (the same way you’d bind a label to its respective input).

<div class="form-field">
	<label for="full-name">Full Name&l;/label>
	<input type="text" id="full-name" name="full-name" required>
	<span data-normform-feedback-for="full-name"></span>
</div>

That’s it. The form will validate as expected, with a few bonus enhancements.

Why NormForm?

  • Progressive enhancement — NormForm encourages proper standards-based markup and attributes as a baseline. HTML5 validation APIs already supply a complete system for handling forms client-side. NormForm leverages this to ensure the form will still run validation in its absence.
  • Improved flow — User experience and conversion improve when providing feedback at just the right time. Form usage studies (such as Baymard’s Usability testing of Inline Validation and Luke W’s excellent Inline Validation in Web Forms) show increases in success and satisfaction when using this approach.
  • HTML-friendly configuration — No need to set up complex validation objects in your JavaScript. Keep validation properties in the markup to make configuration and multi-lingual validation easier.
  • Super lightweight — script clocks in at 2kb minified and gzipped.

Examples

Simple form validation

Out-of-the-box, validate as you go implementation. The normform-invalid class is added to invalid inputs (or their parent if a fieldset of radio or checkbox inputs).

See the Pen NormForm: Simple form validation by Adam Duncan (@adamduncan) on CodePen.

HTML5 input types

HTML5 introduced purpose-built input types like url, date and tel. Use them! NormForm backs up native input type validation.

Pattern matching

Use pattern attribute to validate against particular formats.

Marking items as valid

When a field becomes valid it will have the normform-valid classname applied. Handy for positive feedback styles.

Custom error messages

Add custom field validation messages by setting the data-normform-error-message attribute on the data-normform-feedback-for element.

Format

Error summary

To render a validation summary, add the data-normform-summary attribute to the form element. Set its value to the message you’d like to display before the error list.

Validate on submit

Delay validation by adding data-normform-async="false" to the form element. Validating on submit is the browser’s default behaviour, but it lacks real-time feedback.

Browser support

NormForm uses sensible, modern JavaScript APIs.

NodeList and ES5 Array methods (querySelectorAll, map, filter) help with handling collections. The Constraint Validation API is the real hero here, which receives support from:

  • Chrome 10+
  • Firefox 4+
  • Edge 12+
  • Opera 10.1+
  • Safari 5+
  • Internet Explorer 10+
  • Android Browser 4+
  • iOS Safari 4.1+

Looking to support older browsers? The best option would be to polyfill these bits with es5-shim and H5F. This example of how to include legacy support for NormForm will get you up and running.
