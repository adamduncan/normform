Install

NPM

Install from npm: npm install normform --save

NormForm will be available to import into your project using ES2015 module syntax:

import normform from 'normform';

Download

Grab a distribution copy of normform.min.js and reference it before the closing <body> tag at the end of your document:

<script src="/path/to/normform.min.js" async><script>

Usage

NormForm is self-initialising, so no need to do any scripting tomfoolery beyond the above.

After authoring a top-class HTML form (with the appropriate tags and attributes), add the NormForm class to the form tag:

<form class="js-normform">
	...
</form>

Alongside each required form input, include an empty <span> element, which will become the container to hold the validation feedback. Bind this to the relevant input by including the data-normform-feedback-for attribute, with its value set to match the input id. The same way you’d bind a label to its respective input.

<div class="form-field">
	<label for="full-name">Full Name&l;/label>
	<input type="text" id="full-name" name="full-name" required>
	<span data-normform-feedback-for="full-name"></span>
</div>

That’s it. The form will validate as expected, with a few bonus enhancements.

Why NormForm?

  • Progressive enhancement — NormForm encourages proper standards-based markup and attributes as a baseline. HTML5 validation APIs already supply a complete system for handling forms. NormForm leverages this to ensure...
  • Improved user experience — Studies show form validation and conversion can be improved dramatically by providing feedback at just the right time. [Links to Luke W, A List Apart]
  • HTML-friendly configuration — No need to set up complex validation objects in JavaScript. Keep validation properties in the markup, to make configuration and multi-lingual validation easier.
  • Lightweight — compiled scripts clock in at _kb gzipped and minified.

Examples

Simple form validation

The out-of-the-box implementation. The normform-invalid class will be applied to invalid inputs (or their parent if a fieldset of radio or checkbox inputs).

Language(s)

HTML5 input types

Backed up with native input type validation.

Pattern matching

Use pattern attribute to validate against particular formats.

Marking items as valid

Fields that can be marked as valid will have the normform-valid classname applied, if positive feedback styles are required.

Custom error messages

Add custom field validation messages by setting the data-normform-error-message attribute on the data-normform-feedback-for element.

Format

Error summary

Create a validation summary by adding a data-normform-summary attribute to the form element. Set its value to a message you’d like to display before the error list.

Validate on submit

Delay validation of form inputs until submit button is clicked by adding data-normform-async="false" to the form element. This is a browser’s default behaviour.

Extending: Matching multiple inputs (coming soon)

Extend behaviour by writing custom validations that leverage NormForm validate() and isValid() methods.

Browser support

NormForm is built on top of sensible, modern JavaScript APIs.

NodeList and ES5 Array methods (querySelectorAll, map, filter) help with handling collections. The Constraint Validation API is the real hero here, which recieves support from:

  • Chrome 10+
  • Firefox 4+
  • Edge 12+
  • Opera 10.1+
  • Safari 5+
  • Internet Explorer 10+
  • Android Browser 4+
  • iOS Safari 4.1+

If you’re looking to support older browsers, the best option would be to polyfill these bits with H5F. This example of how to include legacy support for NormForm will get you up and running.
